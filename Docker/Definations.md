Docker:-
1. Use for contatinorization 
2. It contains the required dependency to run a particular code/application.
3. No more "It works on my computer thing"
4. Image = Blueprint** (Immutable), **Container = Running instance**
5. Lightweight
7. Portable - Runs same on dev, test, prod, any cloud
8. Microservices friendly - Each service in separate container
9. Version control for environments - Docker images like code
10. Scaling easy - Multiple containers from same image


Virtual Machines 
1. VM: Guest OS + App = Heavy, slow
2. Its like a computer with in a computer.
3. VM are  created and managed by virtulization software(VM ware, Virtual box etc)
4. Need the entire copy of window,Ubuntu or any other OS even if less component if that OS is required.
5. Full OS boot time is required to start.
6. Resourse Utilization is more (RAM,CPU)

Docker
1. Easy and light weight.
2. Only need the required components.
3. Need less time required to run.
4. Resourse Utilization is less (RAM,CPU)
5. Scalability: Easy to scale by creating more containers.
